Ето най-важните Git команди, подредени според работния процес. Можете да намерите официалната документация за всяка от тях на git-scm.com. 
1. Настройка и Инициализация
git init: Създава ново локално хранилище (repository) в текущата папка.
git clone <url>: Копира съществуващо отдалечено хранилище на вашия компютър.
git config --global user.name "Име": Задава вашето име за всички коммити.
git config --global user.email "email@example.com": Задава вашия имейл адрес. 
2. Ежедневна работа (Промени и Коммити)
git status: Показва състоянието на работната директория (кои файлове са променени или добавени).
git add <file>: Добавя конкретен файл към „staging area“ (подготовка за запис).
git add .: Добавя всички променени файлове наведнъж.
git commit -m "съобщение": Записва подготвените промени в историята с кратко описание.
git diff: Показва разликите между работната директория и последното състояние. 
3. Разклонения (Branching) и Сливане 
git branch: Списък на всички локални клонове.
git switch -c <име>: Създава нов клон и превключва към него (модерна алтернатива на checkout -b).
git checkout <име>: Превключва към съществуващ клон.
git merge <име>: Слива посочения клон в текущия. 
4. Синхронизация с отдалечени хранилища (Remote) 
git push origin <branch>: Изпраща локалните промени към сървъра.
git pull: Изтегля и автоматично слива промените от сървъра в локалното копие.
git fetch: Изтегля информация за промените от сървъра, но без да ги слива веднага. 
5. История и Отмяна на промени
git log: Показва историята на всички направени записи (commits).
git reset --hard HEAD~1: Отменя последния коммит и изтрива всички промени (внимавайте!).
git restore <file>: Отхвърля промените в конкретен файл и го връща към последното записано състояние.
git stash: Временно "скрива" текущите незаписани промени, за да изчисти работната директория.
